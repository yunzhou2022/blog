# spec coding

让AI根据prd，创建spec.md，然后根据spec.md进行编码。

## 什么是spec coding

spec coding（specification-driven coding，规格驱动开发），是指先由AI或人根据产品需求文档（PRD）生成详细的规范说明文档（spec.md），再严格按照该文档进行开发和实现。这样做可以确保开发流程标准化，减少沟通成本，提高产出的一致性和可维护性。

## 用5岁小孩能懂的说法

让AI先写一份很详细的计划书，然后我们照着这个计划一步一步做事情，这样可以让大家更容易明白要做什么，做出来的东西也不会出错。

## 使用speccoding的SOP

1. 阅读和理解产品需求文档（PRD）。
2. 让AI或团队成员根据PRD生成详细的spec.md文档，包含所有功能和细节说明。
3. 围绕spec.md文档进行讨论，完善并确认所有需求细节。
4. 开发过程中，严格按照spec.md逐条实现功能。
5. 过程中如有变更，先更新spec.md，再进行编码调整。
6. 开发完成后，对照spec.md进行自测和验收，确保无遗漏。
7. 若遇到歧义或补充需求，及时回到spec.md补充再继续开发。

## 示例 spec.md

```markdown
# 登录页面规格说明（spec.md 示例）

## 1. 概述
实现一个带有用户名和密码输入框的登录页面，包含登录按钮及表单校验。

## 2. 功能需求
- 用户可以输入用户名和密码
- 用户名和密码均为必填
- 点击登录按钮，如果输入有效则触发登录请求
- 提交时如有校验错误，显示对应错误提示
- 登录请求过程中按钮禁用，加载中
- 登录成功跳转至首页，失败显示提示

## 3. 交互细节
- 输入框未填写时点击登录，分别在下方展示"请输入用户名"、"请输入密码"
- 登录按钮loading状态时文字变为“登录中...”
- 错误提示展示于按钮下方

## 4. UI 规范
- 页面居中显示，宽度 360px
- 按钮主色 #1890ff
- 错误文字颜色 #ff4d4f

## 5. 其它需求
- 所有文案需支持国际化
- 支持 Tab 键切换输入框和按钮
```
