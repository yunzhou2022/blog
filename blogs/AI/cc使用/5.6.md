# Claude Code 进阶使用

## subagent

### 什么是 subagent

subagent 是 Claude Code 中的子代理机制，允许主对话将特定任务委托给一个独立运行的"子对话"来完成。每个 subagent 拥有独立的上下文，执行完毕后将结果返回给主线程。

### subagent 的作用

- **隔离上下文**：把大量中间结果（如大规模代码搜索、日志分析）放在子对话里，避免污染主上下文窗口
- **并行执行**：多个 subagent 可以同时运行，提升处理效率
- **专项处理**：针对特定领域（代码探索、安全审查、文档生成）使用专用的 agent 类型

### 什么时候用

- 任务需要大量文件搜索或代码分析，结果不需要全部保留在主上下文
- 多个独立子任务可以并行完成
- 需要对某段代码做独立评审，避免受当前会话已有结论影响

### 怎么用

在 Claude Code 中，可以通过 `Agent` 工具触发 subagent：

```
# 示例：探索型 subagent
Agent(subagent_type="Explore", prompt="找出所有 API 路由定义")

# 示例：通用 subagent 并行执行
Agent(prompt="分析认证模块安全性", run_in_background=true)
```

## skills

### 什么是 skills

skills（技能）是可安装的、可复用的提示词模块，通过 `/skill-name` 斜杠命令触发。每个 skill 封装了一类专项能力，如代码审查、文档生成、PR 管理等。

### skills 的作用

- **能力扩展**：为 Claude Code 添加项目或团队专属的工作流
- **标准化操作**：将常见操作固化为一条命令，减少重复提示词编写
- **团队共享**：skill 文件可以提交到仓库，团队成员共用同一套工作流

### 什么时候用

- 某类操作需要反复执行（如每次提 PR 都要做安全检查）
- 团队有内部平台需要与 Claude Code 集成（如内部 wiki、CI 系统）
- 想把复杂的多步骤提示词封装成一个简单命令

### 怎么用

1. 在 `.claude/commands/` 目录下创建 Markdown 文件，文件名即命令名
2. 文件内容描述该技能的执行逻辑
3. 在对话中用 `/文件名` 触发

```
# .claude/commands/security-check.md
对当前分支的变更做安全审查，重点关注 SQL 注入、XSS、权限校验...
```

## hooks

### 什么是 hooks

hooks 是在特定事件发生时自动执行的 shell 命令，配置在 `settings.json` 中。类似于 git hooks，但触发点是 Claude Code 的工具调用事件。

### hooks 的作用

- **自动化流程**：在 Claude 写完代码后自动运行格式化、lint、测试
- **审计日志**：记录每次工具调用，方便追溯
- **安全拦截**：在危险命令执行前做额外确认或拦截

### 什么时候用

- 希望"每次 Claude 修改文件后自动运行 prettier"
- 需要在 Claude 执行命令前记录日志
- 想对特定工具调用添加前置/后置操作

### 怎么用

在 `.claude/settings.json` 中配置：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "pnpm lint --fix"
          }
        ]
      }
    ]
  }
}
```

常见事件：`PreToolUse`、`PostToolUse`、`Stop`、`Notification`

## headless

### 什么是 headless

headless 模式是指以非交互方式运行 Claude Code，不需要用户在终端实时输入，适合集成到脚本、CI/CD 流程中自动化执行。

### headless 的作用

- **CI 集成**：在 GitHub Actions 等流水线中调用 Claude Code 完成代码审查、自动修复
- **批量处理**：对多个文件或仓库批量执行同一任务
- **定时任务**：配合 cron 定期运行代码质量检查

### 什么时候用

- 需要在无人值守的环境中运行 Claude Code
- 想把 Claude Code 作为 CI 流程的一个步骤
- 需要脚本化、程序化地调用 Claude Code

### 怎么用

```bash
# 通过 -p 参数传入提示词，--output-format 指定输出格式
claude -p "审查 src/ 目录下所有文件的安全问题，输出报告" \
  --output-format json \
  --no-interactive
```

## plugins

### 什么是 plugins

plugins（插件）是通过 MCP（Model Context Protocol）协议接入的外部工具服务，为 Claude Code 提供访问外部系统的能力，如数据库、浏览器、第三方 API 等。

### plugins 的作用

- **能力扩展**：让 Claude Code 能直接操作浏览器、查询数据库、调用内部服务
- **上下文注入**：从外部系统实时拉取数据作为上下文
- **工具集成**：将公司内部工具（wiki、监控、代码平台）接入 Claude Code

### 什么时候用

- 需要让 Claude Code 访问浏览器（自动化测试、截图）
- 需要连接数据库做数据分析
- 需要与内部系统（如 Jira、Confluence、内部 API）交互

### 怎么用

在 `settings.json` 中配置 MCP server：

```json
{
  "mcpServers": {
    "browser": {
      "command": "npx",
      "args": ["@anthropic-ai/mcp-server-puppeteer"]
    },
    "database": {
      "command": "node",
      "args": ["./mcp-servers/db-server.js"],
      "env": {
        "DB_URL": "postgresql://localhost/mydb"
      }
    }
  }
}
```

配置后，对应工具会自动出现在 Claude Code 的可用工具列表中。
