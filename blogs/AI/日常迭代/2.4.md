# claudecode 介绍

claudecode是一个可编程， 可拓展，可组合的AI Agent 框架。

不止是一个工具，而是一个平台，可以构建自己的工作流。

## 被动使用与主动使用

被动：

`用户 -》 输入问题 -》 claude 回答 -》 结束 `

主动：

`用户-》配置Agent -》 Agent自动工作 -》 自动完成任务 `

| 场景     | 被动用法                   | 主动用法                                        |
| -------- | -------------------------- | ----------------------------------------------- |
| 代码审查 | 帮我看下这个代码有没有问题 | 配置一个code-review sub agent， 每次pr自动审查  |
| 跑测试   | 帮我跑下测试               | 配置Hook，每次修改代码自动跑测试                |
| 写文档   | 帮我写个README             | 配置Skill，Claude识别文档需求时自动应用文档规范 |

类比：

- 使用者： 知道方向盘怎么打，油门动， 刹车停
- 驾驭者： 理解发动机，变速箱，刹车系统的工作， 能够改装车辆

对于 claude code：

- 使用者：知道怎么提问， 怎么让claude帮你写代码
- 驾驭者：理解记忆系统，子代理，技能包，hooks 的原理，能够见自定义工作流

## claude code 底层技术全景图

```
agent sdk 编程接口层
  └─ headless & MCP 集成层
      └─ commands, skills, subAgents, hooks 扩展层
          └─ memory 基础层
```

**各层说明：**

- **memory 基础层**：最基础的一层，负责记录和管理AI的“记忆”，实现信息的存储与检索。为什么需要它？因为只有拥有记忆能力，AI才能理解上下文、追踪历史并做出相关决策，否则每次交互都是“失忆”的。
- **commands, skills, subAgents, hooks 扩展层**：建立在记忆层之上，提供自定义命令、技能、子代理和自动化钩子的能力。作用是让AI变得可编程和可拓展，为什么有这层？因为仅有记忆不足以满足复杂的工作流和多样化需求，通过扩展点可以灵活地适配不同场景。
- **headless & MCP 集成层**：这一层的作用是实现与外部系统、平台的集成，包括无界面（headless）运行和多能力平台（MCP）对接。为什么需要它？只有这样，AI才能自动化地连接外界服务，成为工作流中的“桥梁”，实现规模化协作和数据流转。
- **agent sdk 编程接口层**：最外层，对开发者友好地开放API，让用户可以自定义Agent、对接工具、编排复杂流程。作用是便于扩展和创新，为什么设计这层？因为它决定了整个系统的可定制性与开发效率，是AI平台对外赋能的窗口。


整个框架由上至下依次抽象，底层(memory)支撑能力扩展层，再通过集成层对接各类系统，最上层由SDK对外统一输出能力。
