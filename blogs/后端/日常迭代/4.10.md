# nosql

## 什么是nosql

非关系型数据库，不遵循表结构，更灵活，适合海量数据，高并发、快速扩展。


## nosql的优点

- schema灵活，不用提前建表
- 横向拓展强， 容易加机器抗高并发
- 读写速度快，适合大数据，高并发场景
- 通常不严格支持事务

## nosql 常见类型

- key value 的键值形

像字典、缓存， key 对应value
例： Redis，Memcached

- 文档数据库

存 JSON、BSON 结构，结构灵活
例： MongoDB

- 列族数据库

按列存储， 适合海量数据查询

例： HBase、Cassandra

- 图形数据库

存储节点与关系， 适合社交、推荐

例如： Neo4J

## 什么时候用

- 数据量大， 高并发
- 数据结构经常变化
- 需要快速开发， 快速迭代
- 对强事务要求不高

## 什么是横向拓展与纵向拓展

- 横向

加更多的机器

- 纵向

换更强的机器
